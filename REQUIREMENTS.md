# Filer - 要件定義書

## 1. プロジェクト概要

### 1.1 目的
Windows向けの美しく、ファイル整理が気持ちいいタブ型ファイラーを開発する。
直感的な操作で日常のファイル管理を効率化し、「このファイラー違う」と思わせる体験を提供する。

### 1.2 ターゲットユーザー
- Windowsユーザー
- キーボード操作を好むパワーユーザー
- エクスプローラーの機能に不満を持つユーザー
- ファイル整理を気持ちよく行いたいユーザー

### 1.3 技術スタック
| 分類 | 技術 | 理由 |
|------|------|------|
| バックエンド | Rust + Tauri v2 | 軽量・高速・安全 |
| フロントエンド | React 19 + TypeScript | 豊富なエコシステム・型安全 |
| ビルド | Vite | 高速HMR |
| スタイリング | Tailwind CSS v4 | ユーティリティファースト |
| 状態管理 | Zustand | 軽量・シンプル |

## 2. 機能要件

### 2.0 実装済み機能

#### 2.0.1 タブ型レイアウト
- ブラウザ風タブでフォルダを管理
- タブの追加（Ctrl+T）/ 閉じる（Ctrl+W）/ 切り替え（Ctrl+Tab）
- ミドルクリックでタブを閉じる

#### 2.0.2 ファイル一覧表示
- ファイル名、サイズ、更新日時、種類の表示
- アイコン表示（ファイル種別に応じた50種類以上対応）
- ソート機能（名前、サイズ、日付、種類）
- 隠しファイルの表示/非表示トグル（Ctrl+H）

#### 2.0.3 基本ファイル操作
- ナビゲーション: ディレクトリの移動（Enter / ダブルクリック / Backspace）
- コピー/カット/ペースト（Ctrl+C / Ctrl+X / Ctrl+V）
- 削除（Delete）→ ゴミ箱へ移動
- リネーム（F2）→ インライン編集
- 新規フォルダ/ファイル作成（コンテキストメニュー）
- 選択: 単一/複数（Space / Ctrl+Click / Shift+Click）、全選択（Ctrl+A）
- ドラッグ&ドロップによるファイル移動

#### 2.0.4 ナビゲーション
- アドレスバー（パンくずリスト形式 + 直接入力）
- 戻る/進む（Alt+Left/Right）/ 上へ
- Quick Access（デスクトップ、ドキュメント、ダウンロード等）
- ドライブ一覧

#### 2.0.5 仮置きシェルフ（Stack）
- サイドバーのシェルフにファイルを一時置き（Ctrl+S / D&D）
- 別タブで移動先に移動後、シェルフからペースト（Ctrl+Shift+V）

#### 2.0.6 プレビューパネル
- 画像プレビュー（PNG, JPG, GIF, SVG, WebP等）
- テキストファイルプレビュー
- ステータスバーからのトグル表示

#### 2.0.7 コンテキストメニュー
- 右クリックメニュー（開く、コピー、カット、ペースト、削除、リネーム等）
- プロパティダイアログ（タイムスタンプ、属性、フォルダサイズ計算）

### 2.1 Phase 1 — 基盤＆低コスト高リターン

> シェルフの次に入れるべき、コア体験を固める機能群。
> 実装コストが比較的低く、触った瞬間に「このファイラー違う」と思わせるもの。

#### 2.1.1 Undo/Redo スタック
- ファイル移動・リネーム・削除を全て履歴管理
- Ctrl+Z で即戻し、Ctrl+Y で再実行
- 操作履歴の表示

#### 2.1.2 ファイル on ファイルでフォルダ化
- ファイルを別ファイルにドラッグすると自動でフォルダ作成（iOS風）
- フォルダ名の自動提案

#### 2.1.3 インラインリネーム進化
- リネーム時に拡張子を除外して自動選択
- Tabキーで次のファイルへ連続リネーム

#### 2.1.4 ブックマークバー
- タブ下によく使うフォルダを並べて一発ジャンプ
- ドラッグで並び替え
- ブックマークの追加・削除

#### 2.1.5 クイックルック (Space)
- Spaceキーで画像・テキスト・PDF・動画をオーバーレイプレビュー
- macOS風のモーダルプレビュー
- 矢印キーで前後のファイルに移動

#### 2.1.6 ドラッグ時スタック表示
- 複数ファイルドラッグ時にカードがファン状に重なる表示
- 「+N」バッジで選択数を表示

### 2.2 Phase 2 — パワーユーザー体験

> 操作効率を一段上げる機能。ここでヘビーユーザーが手放せなくなる。

#### 2.2.1 コマンドパレット (Ctrl+K)
- ファイル名あいまい検索
- コマンド実行
- ブックマークジャンプを統合

#### 2.2.2 ワークスペース保存
- 開いてるタブの状態をまるごと保存・復元
- 複数ワークスペースの管理

#### 2.2.3 コピーキュー
- コピー/移動をキューで順次実行
- 進捗表示・一時停止・キャンセル対応

#### 2.2.4 テンプレートフォルダ
- 事前定義したフォルダ構成をワンクリック生成

#### 2.2.5 クリップボードからファイル生成
- Ctrl+Vでクリップボード内の画像/テキストをその場でファイル化

### 2.3 Phase 3 — キラー機能（差別化の核）

> 他のファイラーにはない、独自の差別化機能。

#### 2.3.1 ドラッグ時の移動先サジェスト
- ドラッグ開始時に移動先候補をIntelliSense風にポップアップ
- 過去の移動履歴・ファイル種別・ブックマークから推測

#### 2.3.2 フォルダルール（自動化）
- 「このフォルダに入ったファイルを自動で○○する」ルールをGUIで設定

#### 2.3.3 Rules.md + AI自動整理
- 自然言語でルールを記述 → AIが解釈してルールエンジンに変換
- ドライラン → 確認 → 実行のフロー

#### 2.3.4 スマートフォルダ（仮想フォルダ）
- 検索条件を保存して仮想フォルダとして扱う

### 2.4 Phase 4 — 磨き上げ

#### 2.4.1 ソート切替アニメーション
- ソート変更時に各ファイルがスムーズに移動するアニメーション

#### 2.4.2 フォルダの中身チラ見せ
- フォルダアイコンにホバーで中身の画像を4枚タイルでプレビュー

#### 2.4.3 カスタムアクション
- 右クリックにスクリプト定義のオリジナル操作を追加

#### 2.4.4 整理スコア
- フォルダの散らかり度を可視化

#### 2.4.5 重複ファイル検出
- 同一ハッシュのファイルを検出 → 削除提案

## 3. 非機能要件

### 3.1 パフォーマンス
- ディレクトリ表示: 1万ファイルを500ms以内に表示
- ファイル検索: インクリメンタル検索のレスポンスは100ms以内
- メモリ使用量: アイドル時100MB以下
- 起動時間: 2秒以内

### 3.2 安全性
- ファイル削除は常に確認ダイアログを表示
- ゴミ箱への移動をデフォルト動作とする
- Undo/Redoによる操作の取り消し対応（Ctrl+Z / Ctrl+Y）

### 3.3 ユーザビリティ
- 直感的なUIで初見でも基本操作が可能
- ツールチップによるキーボードショートカットの表示
- 美しいアニメーションと視覚的フィードバック

### 3.4 拡張性
- モジュラー設計による機能追加の容易さ
- 設定ファイル（JSON/TOML）による挙動カスタマイズ
- Tauri IPC を通じた明確なフロント/バック分離

## 5. データ設計

### 5.1 設定ファイル
- 保存場所: `%APPDATA%/filer/config.toml`
- キーバインド設定: `%APPDATA%/filer/keybindings.toml`
- ブックマーク: `%APPDATA%/filer/bookmarks.json`

### 5.2 セッション管理
- 前回終了時のタブ状態を保存・復元
- ウィンドウサイズ・位置の記憶

### 5.3 操作履歴
- Undo/Redo用の操作ログ（インメモリ + オプションで永続化）
- 移動履歴DB（移動先サジェスト用、Phase 3）
