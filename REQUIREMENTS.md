# Filer - 要件定義書

## 1. プロジェクト概要

### 1.1 目的
Windows向けの高速・軽量なデュアルパネルファイラーを開発する。
キーボード操作中心で、日常のファイル管理作業を効率化する。

### 1.2 ターゲットユーザー
- Windowsユーザー
- キーボード操作を好むパワーユーザー
- エクスプローラーの機能に不満を持つユーザー

### 1.3 技術スタック
| 分類 | 技術 | 理由 |
|------|------|------|
| バックエンド | Rust + Tauri v2 | 軽量・高速・安全 |
| フロントエンド | React 19 + TypeScript | 豊富なエコシステム・型安全 |
| ビルド | Vite | 高速HMR |
| スタイリング | Tailwind CSS v4 | ユーティリティファースト |
| 状態管理 | Zustand | 軽量・シンプル |

## 2. 機能要件

### 2.1 コア機能 (MVP - Phase 1)

#### 2.1.1 デュアルパネル表示
- 左右2つのパネルにディレクトリ内容を表示
- アクティブパネルをハイライト表示
- パネル間のフォーカス切り替え（Tab / Ctrl+Left/Right）
- パネル分割比率の調整（ドラッグまたはキーボード）

#### 2.1.2 ファイル一覧表示
- ファイル名、サイズ、更新日時、種類の表示
- アイコン表示（ファイル種別に応じた）
- ソート機能（名前、サイズ、日付、種類）
- 昇順/降順の切り替え
- 隠しファイルの表示/非表示トグル
- 詳細表示 / アイコン表示の切り替え

#### 2.1.3 基本ファイル操作
- **ナビゲーション**: ディレクトリの移動（Enter / Backspace）
- **コピー**: ファイル/フォルダのコピー（F5 / Ctrl+C）
- **移動**: ファイル/フォルダの移動（F6 / Ctrl+X → Ctrl+V）
- **削除**: ファイル/フォルダの削除（F8 / Delete）→ 確認ダイアログ
- **リネーム**: ファイル/フォルダ名の変更（F2）
- **新規作成**: フォルダ/ファイルの新規作成（F7 / Ctrl+N）
- **選択**: 単一選択 / 複数選択（Space / Ctrl+Click / Shift+Click）
- **全選択**: Ctrl+A
- **選択反転**: Ctrl+I

#### 2.1.4 アドレスバー
- 現在のパスを表示
- パスの直接入力によるジャンプ
- パンくずリスト形式での表示
- パス履歴のドロップダウン

#### 2.1.5 キーボード操作
- Vim風ナビゲーション（j/k で上下移動、h/l でディレクトリ移動）
- コマンドパレット（Ctrl+P）でコマンド検索・実行
- カスタマイズ可能なキーバインド
- キーボードショートカット一覧表示（F1 / ?）

### 2.2 拡張機能 (Phase 2)

#### 2.2.1 タブ機能
- 各パネルで複数タブを開く
- タブの追加（Ctrl+T）/ 閉じる（Ctrl+W）
- タブの並び替え（ドラッグ&ドロップ）
- タブの復元（Ctrl+Shift+T）
- タブにカスタム名を設定

#### 2.2.2 高速検索・フィルタ
- **インクリメンタル検索**: 現在のディレクトリ内でリアルタイム絞り込み（/）
- **ファジー検索**: 曖昧文字列マッチング
- **再帰検索**: サブディレクトリを含む深い検索（Ctrl+F）
- **フィルタ**: 拡張子、サイズ、日付による絞り込み
- **正規表現対応**: 検索パターンに正規表現を使用可能

#### 2.2.3 プレビュー機能
- テキストファイルのプレビュー（シンタックスハイライト付き）
- 画像ファイルのサムネイルプレビュー
- プレビューパネルのトグル表示

#### 2.2.4 ブックマーク
- よく使うディレクトリをブックマーク登録
- ブックマークへのクイックジャンプ（Ctrl+B）
- ブックマークの管理（追加・削除・並び替え）

### 2.3 高度な機能 (Phase 3)

#### 2.3.1 バッチ操作
- 一括リネーム（パターン指定、連番、正規表現置換）
- ファイル比較（左右パネルの内容比較）

#### 2.3.2 アーカイブ対応
- ZIP/7z/tar.gz の中身を透過的に表示
- アーカイブの作成・展開

#### 2.3.3 外部ツール連携
- ターミナルを現在のディレクトリで開く（Ctrl+`）
- 外部エディタでファイルを開く
- カスタムコマンドの登録・実行

#### 2.3.4 テーマ・カスタマイズ
- ダーク/ライトテーマの切り替え
- カスタムテーマの作成
- フォント・フォントサイズの変更
- カラムの表示/非表示・並び替え

## 3. 非機能要件

### 3.1 パフォーマンス
- ディレクトリ表示: 1万ファイルを500ms以内に表示
- ファイル検索: インクリメンタル検索のレスポンスは100ms以内
- メモリ使用量: アイドル時100MB以下
- 起動時間: 2秒以内

### 3.2 安全性
- ファイル削除は常に確認ダイアログを表示
- ゴミ箱への移動をデフォルト動作とする（完全削除はShift+Delete）
- 操作の取り消し（Undo）対応（Ctrl+Z）

### 3.3 ユーザビリティ
- 直感的なUIで初見でも基本操作が可能
- ツールチップによるキーボードショートカットの表示
- 設定画面でのGUI設定変更

### 3.4 拡張性
- プラグインシステムを見据えたモジュラー設計
- 設定ファイル（JSON/TOML）による挙動カスタマイズ
- Tauri IPC を通じた明確なフロント/バック分離

## 4. UI設計方針

### 4.1 レイアウト
```
┌─────────────────────────────────────────────────┐
│ ツールバー: [戻る][進む][上へ] [アドレスバー] [検索] │
├────────────────────┬────────────────────────────┤
│ [Tab1][Tab2][+]    │ [Tab1][Tab2][+]            │
├────────────────────┼────────────────────────────┤
│                    │                            │
│  左パネル           │  右パネル                   │
│  (ファイル一覧)      │  (ファイル一覧)              │
│                    │                            │
│                    │                            │
│                    │                            │
├────────────────────┴────────────────────────────┤
│ ステータスバー: 選択数 | 合計サイズ | 空き容量      │
└─────────────────────────────────────────────────┘
```

### 4.2 カラースキーム
- ダークテーマをデフォルト
- ハイコントラストで視認性を確保
- アクティブパネルを明確に区別

## 5. データ設計

### 5.1 設定ファイル
- 保存場所: `%APPDATA%/filer/config.toml`
- キーバインド設定: `%APPDATA%/filer/keybindings.toml`
- ブックマーク: `%APPDATA%/filer/bookmarks.toml`

### 5.2 セッション管理
- 前回終了時のタブ・パネル状態を保存・復元
- ウィンドウサイズ・位置の記憶
